Two approaches to polish your CRM

Both use the Minimal & Clean monochrome direction you chose

A

Systematic Polish (Recommended)

Introduce CSS variables for consistency, then apply fixes uniformly across all elements. Most reliable, prevents future drift.

What changes:
  • Add ~15 CSS variables (colors, radius, shadows, spacing)
  • Unify border-radius: 6px for cards, 4px for inputs/buttons
  • Add subtle box-shadow to cards, modals, dropdowns
  • Standardize hover states (0.2s ease, consistent feedback)
  • Add active/pressed states to all buttons
  • Clean up z-index scale (10, 20, 30...)
  • Soften borders from harsh #000 to #e0e0e0/#ddd

Pros

  • Consistent, maintainable
  • Future changes are one-line edits
  • Catches all issues

Cons

  • More CSS changes overall
  • Takes slightly longer
B

Targeted Fixes

Fix only the visible inconsistencies directly — no CSS variables, just patch the problems where they are.

What changes:
  • Add border-radius to buttons, inputs, modals inline
  • Add shadows to cards and modals inline
  • Standardize hover transitions where broken
  • Fix the worst z-index conflicts

Pros

  • Smaller change set
  • Quicker to apply

Cons

  • Problems can creep back
  • Less maintainable
  • May miss some issues
